================================================================================
                        HOW IT WORKS NOW
================================================================================

FLOW: Get Question → Answer → Get Next Question

Step 1: User Gets First Question
GET /chat/question
↓
get_question() loads project
↓
Calls orchestrator._generate_questions_deduplicated()
↓
Question tracked in asked_questions (status: pending)
↓
Returns unique question

Step 2: User Answers Question
POST /chat/message "Addition, subtraction..."
↓
Orchestrator processes response
↓
Updates asked_questions entry:
  - Sets answer field
  - Records timestamp
  - Changes status to "answered"
↓
Saves to project
↓
Returns response

Step 3: User Gets Next Question
GET /chat/question
↓
_generate_questions_deduplicated() called
↓
Filters against asked_questions + skipped_questions
↓
Fuzzy matching prevents similar questions (70% threshold)
↓
Returns NEW question on DIFFERENT topic
↓
User: "Finally! A new question!"
