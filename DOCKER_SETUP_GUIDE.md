# Socrates Docker Setup Guide

**Complete guide for setting up, upgrading, and troubleshooting Socrates in Docker.**

---

## Quick Start (First Time)

```bash
git clone https://github.com/Nireus79/Socrates.git
cd Socrates

# Generate encryption keys
./setup-secrets.sh

# Start Docker with live logs
sudo docker compose --env-file .env.local up --build
```

Then open: **http://localhost:3000**

## Clean Rebuild (Upgrading or Fresh Install)

If you're upgrading Socrates or starting fresh after an old installation:

### Step 1: Stop and Clean

```bash
cd ~/PycharmProjects/Socrates

# Stop all containers
sudo docker compose down

# Delete old volume (contains old database)
sudo docker volume rm socrates_data

# Clean up old images to save disk space
sudo docker image prune -a -f
sudo docker builder prune -a -f

# View space recovered
sudo docker system df
```

### Step 2: Generate New Encryption Keys

```bash
# Generate fresh encryption keys (one-time per environment)
./setup-secrets.sh

# Verify keys were created
ls -l .env.local
```

### Step 3: Rebuild and Monitor Logs

```bash
# Rebuild from scratch (with live logs on terminal)
sudo docker compose --env-file .env.local up --build
```

**Watch the logs** - You should see:
```
socrates-api       | INFO:     Uvicorn running on http://0.0.0.0:8000
socrates-frontend  | ✓ compiled successfully
nginx             | 2026-07-08 ... started
```

### Step 4: Verify in Another Terminal

While Docker is running, open a **new terminal**:

```bash
# Check API health
curl http://localhost:8000/health

# Should respond: {"status":"ok"}

# Check frontend is accessible
curl http://localhost:3000

# Should respond with HTML
```

### Step 5: Configure LLM API Keys

1. Open browser: **http://localhost:3000**
2. Register or login
3. Go to **Settings → LLM**
4. Add your API keys:
   - Claude (Anthropic API key)
   - Ollama (if using local Ollama)
   - OpenAI (if using OpenAI)
   - Gemini (if using Google)

## Restart After Closing

### Simple Restart (Containers Still Exist)

```bash
# Just start the containers again (fast)
sudo docker compose --env-file .env.local up
```

### Full Restart (Clean State)

```bash
# Stop and remove containers, but keep database
sudo docker compose down

# Restart fresh
sudo docker compose --env-file .env.local up
```

### Complete Reset (Delete Database)

```bash
# Stop containers and delete volume (loses all data)
sudo docker compose down -v

# Rebuild completely fresh
sudo docker compose --env-file .env.local up --build
```

## Encryption Keys & Security

### What is `.env.local`?

- **File**: Contains 3 encryption keys generated by `setup-secrets.sh`
- **Location**: Root of Socrates repository
- **Git**: In `.gitignore` - NEVER committed to version control
- **Purpose**: 
  - `SOCRATES_ENCRYPTION_KEY` - Encrypts API credentials
  - `DATABASE_ENCRYPTION_KEY` - Encrypts database
  - `JWT_SECRET_KEY` - Signs session tokens

### How Keys Work

1. `setup-secrets.sh` generates random 32-byte keys
2. Keys saved to `.env.local` with restricted permissions (0600)
3. Docker reads keys from `.env.local` via `--env-file` flag
4. All API keys encrypted before storing in database
5. Same key used for decryption when retrieving

### Rotating Keys (Change Encryption)

```bash
# Generate NEW encryption keys
./setup-secrets.sh

# Restart Docker (will use new keys for NEW credentials)
sudo docker compose down
sudo docker compose --env-file .env.local up --build

# Set your LLM API keys again in Settings
```

**Important**: Old API keys won't decrypt with new keys (expected)
- Users must re-enter their API keys after rotation
- This is a security feature (forces key verification)

### Production Deployment

For production (CI/CD, Kubernetes, Cloud):

1. **Generate keys ONCE locally**:
   ```bash
   ./setup-secrets.sh
   ```

2. **Store keys in secret manager**:
   - GitHub Actions Secrets
   - Kubernetes Secrets
   - AWS Secrets Manager
   - HashiCorp Vault
   - Docker Swarm Secrets

3. **Pass keys at deploy time** (never in docker-compose.yml):
   ```bash
   export SOCRATES_ENCRYPTION_KEY=<from-secrets-manager>
   export DATABASE_ENCRYPTION_KEY=<from-secrets-manager>
   export JWT_SECRET_KEY=<from-secrets-manager>
   docker compose up --build
   ```

## Docker Compose Commands

### Basic Operations

```bash
# Start services (attach to logs)
sudo docker compose --env-file .env.local up

# Start in background
sudo docker compose --env-file .env.local up -d

# Stop services
sudo docker compose down

# Stop and remove volumes (delete database!)
sudo docker compose down -v

# Rebuild images
sudo docker compose --env-file .env.local up --build

# View running services
sudo docker compose ps

# View service status
sudo docker compose ps --all
```

### Logs

```bash
# View logs for all services
sudo docker compose logs

# Follow logs (live)
sudo docker compose logs -f

# View logs for specific service
sudo docker compose logs -f socrates-api
sudo docker compose logs -f socrates-frontend

# View last N lines
sudo docker compose logs --tail 100

# Show timestamps
sudo docker compose logs -f --timestamps
```

### Maintenance

```bash
# Clean up stopped containers
sudo docker container prune -f

# Delete unused images
sudo docker image prune -a -f

# Delete build cache
sudo docker builder prune -a -f

# Delete unused volumes
sudo docker volume prune -f

# Show disk usage
sudo docker system df

# Show detailed info
sudo docker system df -v
```

## Troubleshooting

### API Fails to Start

**Error**: `SOCRATES_ENCRYPTION_KEY not set`

```bash
# Check if .env.local exists
ls -l .env.local

# If missing, generate keys
./setup-secrets.sh

# Restart Docker
sudo docker compose down
sudo docker compose --env-file .env.local up --build
```

### Database Encryption Error

**Error**: `Failed to decrypt data: Invalid encrypted data format: missing salt`

This happens when upgrading with an old database that has incompatible encryption.

```bash
# Delete old database volume
sudo docker volume rm socrates_data

# Generate new keys
./setup-secrets.sh

# Rebuild fresh
sudo docker compose --env-file .env.local up --build
```

### Port Already in Use

**Error**: `bind: address already in use`

```bash
# Option 1: Stop conflicting Docker service
sudo docker compose down

# Option 2: Change ports in docker-compose.yml
# Edit docker-compose.yml and change:
#   ports:
#     - "3000:80"  <- change first number
#     - "8000:8000" <- change first number

# Then restart
sudo docker compose --env-file .env.local up --build
```

### Container Exits Immediately

```bash
# Check why it exited
sudo docker compose logs socrates-api

# Common causes:
# - Missing .env.local file
# - Wrong --env-file path
# - Invalid PYTHONPATH
# - Missing database file permissions
```

### Can't Connect to Frontend (http://localhost:3000)

```bash
# Check if frontend container is running
sudo docker compose ps socrates-frontend

# Check frontend logs
sudo docker compose logs -f socrates-frontend

# Check nginx (reverse proxy)
sudo docker compose logs -f nginx

# Verify port mapping
sudo docker compose ps

# Try direct port
curl http://localhost:80
```

### API Endpoint Returns Error

```bash
# Check API logs for errors
sudo docker compose logs -f socrates-api

# Common issues:
# - Database file locked (permissions)
# - Missing encryption key
# - Incompatible database schema
```

## Docker Architecture

### Services

**api** (socrates-api)
- FastAPI server on port 8000
- Handles all business logic
- Connects to SQLite database at `/app/data/projects.db`
- Requires `SOCRATES_ENCRYPTION_KEY` environment variable

**frontend** (socrates-frontend)
- React/Vue UI on port 80 (inside container)
- Maps to http://localhost:3000 (externally)
- Communicates with API via `/api` proxy

**nginx** (reverse proxy)
- Routes `/api/*` requests to API service
- Routes `/` to frontend service
- Handles CORS and security headers

**redis** (cache)
- In-memory cache for sessions and data
- Optional, used for performance

### Volumes

**socrates_data**
- Persistent storage for SQLite database
- Survives container restarts
- Located at `/app/data` inside container
- Deleted with `docker compose down -v`

**redis_data**
- Persistent Redis data
- Deleted with `docker compose down -v`

### Networks

**socrates-network**
- Docker bridge network
- Allows services to communicate by name
- api, frontend, nginx, redis all on same network

## Backup and Restore

### Backup Database

```bash
# While Docker is running
docker cp socrates-api:/app/data/projects.db ./projects.db.backup

# Or stop and copy from volume
sudo docker run --rm -v socrates_data:/data -v $(pwd):/backup \
  alpine tar czf /backup/socrates_data.tar.gz -C /data .
```

### Restore Database

```bash
# Stop Docker
sudo docker compose down

# Delete current volume
sudo docker volume rm socrates_data

# Create volume and restore
sudo docker volume create socrates_data
sudo docker run --rm -v socrates_data:/data -v $(pwd):/backup \
  alpine tar xzf /backup/socrates_data.tar.gz -C /data

# Start Docker
sudo docker compose --env-file .env.local up
```

## Notes for Users

- ✅ **Always** use `./setup-secrets.sh` before first Docker run
- ✅ **Always** pass `--env-file .env.local` to docker compose
- ✅ **Never** commit `.env.local` to git
- ✅ **Always** keep `.env.local` secure (in .gitignore)
- ✅ **Always** check logs when something goes wrong
- ✅ **Always** use `docker compose down -v` when doing clean rebuild
- ❌ **Never** hardcode secrets in docker-compose.yml
- ❌ **Never** share .env.local between environments
- ❌ **Never** commit .env files to version control
