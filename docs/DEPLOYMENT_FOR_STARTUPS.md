# Deploying Socrates for Your Startup

> No DevOps experience needed. Get Socrates running on your infrastructure in 1 hour.

---

## TL;DR — Pick Your Platform

| Platform | Cost | Setup Time | Best For |
|----------|------|-----------|----------|
| **Fly.io** | $5-20/mo | 5 min | Fast, simple deployments |
| **Railway** | $10-50/mo | 10 min | GitHub integration |
| **AWS ECS** | $20-100/mo | 30 min | Enterprise, existing AWS |
| **DigitalOcean** | $10-60/mo | 20 min | Full control, affordable |
| **Heroku** | $50-500+/mo | 5 min | Hands-off, pricier |
| **Docker Compose** | Free | 10 min | Local testing, small teams |

**Recommendation for startups:** Start with **Railway** (GitHub + auto-deploy) or **Fly.io** (cheapest).

---

## 1️⃣ Option A: Railway (Recommended for Startups)

**Why:** Auto-deploys on every GitHub push. Zero DevOps. Works great for small teams.

### Step 1: Connect GitHub
1. Go to [railway.app](https://railway.app)
2. Sign up with GitHub
3. Connect to your Socrates repository

### Step 2: Deploy
1. Create a new project
2. Add services:
   - PostgreSQL (click add plugin)
   - Redis (click add plugin)
   - Node/Python (select Node for environment)

### Step 3: Configure Environment
Railway auto-reads your `.env` file. Add these variables:
```bash
DATABASE_URL=postgresql://...  # Auto-generated by Railway
REDIS_URL=redis://...          # Auto-generated by Railway
ANTHROPIC_API_KEY=sk-ant-...
ENVIRONMENT=production
SECRET_KEY=<generate-random-string>
```

### Step 4: Deploy
1. Connect your Socrates repo
2. Railway auto-deploys on every push
3. Your app is live in 10 minutes

**Cost:** $10-50/month depending on usage

## 2️⃣ Option B: Fly.io (Cheapest)

**Why:** Simplest CLI, global CDN, cheapest option, very fast.

### Step 1: Install Fly CLI
```bash
curl -L https://fly.io/install.sh | sh
```

### Step 2: Sign Up
```bash
flyctl auth signup
```

### Step 3: Initialize Socrates
```bash
cd Socrates
flyctl launch --name my-socrates
```

Choose:
- Org: Create new
- PostgreSQL: Yes
- Redis: Yes

### Step 4: Set Secrets
```bash
flyctl secrets set ANTHROPIC_API_KEY=sk-ant-...
flyctl secrets set ENVIRONMENT=production
```

### Step 5: Deploy
```bash
flyctl deploy
```

**Cost:** $5-20/month

**Check status:**
```bash
flyctl status
flyctl logs
```

## 3️⃣ Option C: AWS ECS (For Larger Deployments)

**Why:** Enterprise features, auto-scaling, integrates with AWS.

### Step 1: Prerequisites
```bash
pip install awscli-local
aws configure  # Enter your AWS credentials
```

### Step 2: Use CloudFormation Template
```bash
cd deployment/aws-ecs
aws cloudformation create-stack \
  --stack-name socrates-stack \
  --template-body file://socrates-stack.yaml \
  --parameters ParameterKey=ImageUri,ParameterValue=<docker-image-uri>
```

### Step 3: Wait for Stack Creation
```bash
aws cloudformation wait stack-create-complete --stack-name socrates-stack
```

### Step 4: Get URL
```bash
aws cloudformation describe-stacks \
  --stack-name socrates-stack \
  --query 'Stacks[0].Outputs'
```

**Cost:** $20-100/month depending on usage

## 4️⃣ Option D: Docker Compose (For Local/Testing)

**Why:** Free, works on any machine, good for testing.

### Step 1: Clone Repository
```bash
git clone https://github.com/Nireus79/Socrates.git
cd Socrates
```

### Step 2: Create Environment
```bash
cp deployment/configurations/.env.example .env
# Edit .env and add: ANTHROPIC_API_KEY=sk-ant-...
```

### Step 3: Start Services
```bash
docker-compose -f deployment/docker/docker-compose.yml up -d
```

### Step 4: Access
- Frontend: http://localhost:3000
- API: http://localhost:8000
- API Docs: http://localhost:8000/docs

**Cost:** Free (just your server costs)

---

## 5️⃣ Option E: DigitalOcean App Platform

**Why:** Middle ground between simplicity and control.

### Step 1: Push to GitHub
Make sure your Socrates fork is on GitHub.

### Step 2: Create App
1. Go to [digitalocean.com/products/app-platform](https://digitalocean.com/products/app-platform)
2. Click "Create App"
3. Connect GitHub repository
4. Select `docker-compose.yml`

### Step 3: Configure
DigitalOcean reads your docker-compose file automatically.

### Step 4: Deploy
Click Deploy. App is live in 5 minutes.

**Cost:** $10-60/month

## Post-Deployment Checklist

After deploying to production:

### Security
- [ ] Set `ENVIRONMENT=production`
- [ ] Generate strong `SECRET_KEY`
- [ ] Enable HTTPS/TLS (all platforms do this automatically)
- [ ] Set database connection pooling limits

### Database
- [ ] Enable automated backups (daily minimum)
- [ ] Test backup restoration
- [ ] Set up read replicas if using RDS

### Monitoring
- [ ] Configure health checks (all platforms do this)
- [ ] Set up alerts for high error rates
- [ ] Monitor CPU/memory usage

### Access
- [ ] Share domain with team
- [ ] Create admin accounts
- [ ] Test authentication

## Common Issues & Fixes

### "Connection refused" to Database
```bash
# Check if PostgreSQL is running
docker ps | grep postgres

# Restart services
docker-compose restart
```

### "Out of memory"
Increase service resource limits in your cloud platform's dashboard.

### "Slow API responses"
1. Check Redis is connected: `redis-cli ping`
2. Check database query performance: Enable slow query log
3. Scale up instances (add more CPU/RAM)

### "Missing ANTHROPIC_API_KEY"
```bash
# For Railway/Fly/AWS:
flyctl secrets list  # Verify secrets are set
```

## Scaling Beyond Startup Phase

When you outgrow single-instance deployments:

1. **Add Load Balancer** - Distribute traffic across instances
2. **Database Replicas** - Read replicas for analytics
3. **Redis Cluster** - Distributed caching for scale
