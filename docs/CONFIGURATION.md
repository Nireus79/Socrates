# Socrates Configuration Guide

This guide covers how to configure and initialize Socrates for different environments (development, staging, production).

---

## Quick Start

### For Development

```bash
# 1. Clone the repository
git clone https://github.com/Nireus79/Socrates.git
cd Socrates

# 2. Auto-generate configuration
python setup_env.py

# 3. Start Socrates (choose one)
python socrates.py                    # CLI mode
python socrates.py --api              # API only
python socrates.py --frontend         # Frontend only
python socrates.py --full             # Full-stack (API + Frontend)
```

### For Production

```bash
# 1. Copy production configuration
cp deployment/configurations/.env.production.example .env

# 2. Edit .env with your values
nano .env

# 3. Start with your deployment tool (Docker, systemd, etc.)
```

## Configuration Files

### Root Configuration Files

**`.env` (DO NOT COMMIT)**
- Main environment configuration
- Auto-generated by `setup_env.py` for development
- Contains secrets and deployment-specific values
- See `.env.example` for all available options

**`.env.example` (Version controlled)**
- Template showing all available environment variables
- Documented descriptions and examples
- Copy to `.env` and fill in values

### Deployment-Specific Templates

Located in `deployment/configurations/`:

**`.env.example`**
- Main template with defaults for most development scenarios
- Includes comments and examples

**`.env.local.example`**
- For local secrets not committed to version control
- Can be sourced in addition to `.env`
- Example: personal API keys

**`.env.production.example`**
- Hardened defaults for production
- Security-focused settings
- Stricter validation

**`.env.staging.example`**
- Staging environment configuration
- Testing settings enabled
- Logging more verbose

### Application Configuration Files

**`socratic_system/config/constants.py`**
- Maturity thresholds (MATURITY_THRESHOLD_HIGH, etc.)
- Project phases and their definitions
- Quality gates and progression criteria
- System-wide constants

**`socratic_system/constants.py`**
- Re-exports from socratic_core and local constants
- Unified constants interface

## Environment Variables Reference

### Core Configuration (Required)

**`ENVIRONMENT`**
- Deployment environment: development, staging, production
- Default: Auto-detected from hostname
- Impact: Affects CORS, security headers, logging

**`JWT_SECRET_KEY`**
- Secret for signing JWT authentication tokens
- **REQUIRED** for API authentication
- Must be ≥32 characters in production
- Auto-generated by `setup_env.py` if missing
- **Never commit to version control**

**`ANTHROPIC_API_KEY`**
- API key for Claude LLM access
- Get from: https://console.anthropic.com/
- Required for: Code generation, analysis, learning
- Optional for: Basic operations
- Leave empty for offline mode

### Server Configuration

**`SOCRATES_API_HOST`**
- Bind address for API server
- Default: 127.0.0.1 (localhost only)
- Use 0.0.0.0 for remote connections
- Don't expose directly to internet (use reverse proxy)

**`SOCRATES_API_PORT`**
- API server port
- Default: 8000
- Auto-detected if port conflicts on startup

**`SOCRATES_API_RELOAD`**
- Enable auto-reload on file changes
- Default: false
- Only for development (disables in production)

**`FRONTEND_HOST` & `FRONTEND_PORT`**
- Frontend development server configuration
- Default: 127.0.0.1:3000
- Only used in `--full` and `--frontend` modes

### Data & Storage

**`SOCRATES_DATA_DIR`**
- Directory for local data storage
- Default: ~/.socrates
- Contains: SQLite databases, cache files
- Must be writable by Socrates process
- Can be on different disk for performance

### Database Configuration (Optional)

**`DATABASE_URL`**
- PostgreSQL connection string (for future migration)
- Format: postgresql://user:password@host:port/database
- If not set: Uses SQLite
- Enables scalable architecture

**`DATABASE_REPLICA_URLS`**
- Read replicas for analytics/reporting
- Comma-separated list of connection strings
- Only used if DATABASE_URL is set
- Example: url1,url2,url3

**`DATABASE_ENCRYPTION_KEY`**
- Encryption key for sensitive database fields
- Optional: Only if SECURITY_DATABASE_ENCRYPTION=true
- 32 character minimum

**`SECURITY_DATABASE_ENCRYPTION`**
- Enable field-level encryption
- Default: false
- Impacts performance (use only if needed)

### Caching & Performance

**`REDIS_URL`**
- Redis connection string
- Default: redis://redis:6379/0
- Used for: Rate limiting, caching
- Gracefully degrades if unavailable
- Can run without Redis (slower)

**`ENABLE_QUERY_PROFILING`**
- Enable database query profiling
- Default: true (development), false (production)
- Performance impact: ~10%

### Feature Flags

**`ENABLE_RATE_LIMITING`**
- Enable request rate limiting
- Default: true
- Limits: 5/min (auth), 100/min (API)
- Requires Redis

**`ENABLE_METRICS`**
- Enable Prometheus metrics
- Default: true
- Metrics endpoint: /metrics
- Low performance impact

**`ENABLE_WEBSOCKET`**
- Enable WebSocket support
- Default: true
- For real-time updates

### CORS & Security

**`ALLOWED_ORIGINS`**
- Comma-separated CORS allowed origins
- Default: localhost variants
- Production: Restrict to your domain
- Examples: http://localhost:3000,https://example.com

### Logging

**`LOG_LEVEL`**
- Logging verbosity
- Values: DEBUG, INFO, WARNING, ERROR, CRITICAL
- Default: INFO (development), WARNING (production)

**`DEBUG`**
- Enable debug mode
- Default: false
- Extra logging, slower performance

## Initialization Sequence

### Phase 1: Environment Setup

1. **Virtual environment check** (socrates.py:33-50)
   - Verifies running in virtual environment
   - Re-executes with venv Python if needed

2. **Add current directory to sys.path** (socrates.py:61-63)
   - Ensures local code takes precedence
   - Allows importing from current directory

3. **Load .env file** (socrates.py:69-86)
   - Reads .env from current directory
   - Manual parsing (not using dotenv)
   - Only sets variables not already set

4. **Import core modules** (socrates.py:52-53)
   - socratic_core (from PyPI)
   - socratic_system.orchestration (local)

### Phase 2: API Initialization (If --api mode)

1. **Load environment variables** (main.py:22)
   - dotenv.load_dotenv() called first
   - Before any config-dependent imports

2. **Initialize application** (main.py:273-281)
   - Create FastAPI app
   - Register lifespan handler

3. **Configure security & CORS** (main.py:285-365)
   - Detect environment
   - Setup CORS based on ALLOWED_ORIGINS
   - Add security headers
   - Enable audit logging

4. **Register routers** (main.py:392-445)
   - Include 30+ API routers
   - Handle errors gracefully
   - Compile Starlette app

5. **Start Uvicorn server** (main.py:461)
   - Listen on SOCRATES_API_HOST:SOCRATES_API_PORT
   - Ready for requests

### Phase 3: Orchestrator Initialization

**Lazy initialization** - happens on first request:

1. Check if orchestrator already created
2. Get ANTHROPIC_API_KEY from environment
3. Create SocraticLibraryManager
4. Initialize agents with LLMClient
5. Setup PureOrchestrator with callbacks
6. Store in app_state for reuse

**Why lazy?**
- Server starts faster
- Don't initialize agents if not used
- Can set API key after startup

### Phase 4: Database Initialization

**On-demand initialization**:

1. Connection pool creates SQLite connection
2. Tables created if not exist
3. Schema migrations run if needed
4. Respects SOCRATES_DATA_DIR

**Multiple databases**:
- `api_projects.db` - API layer (FastAPI)
- `projects.db` - Core system (business logic)

## Configuration by Environment

### Development Configuration

```bash
ENVIRONMENT=development
JWT_SECRET_KEY=dev-secret-key-min-32-chars-required
ANTHROPIC_API_KEY=sk-ant-...  # Or leave empty
SOCRATES_API_HOST=127.0.0.1
SOCRATES_API_PORT=8000
SOCRATES_DATA_DIR=~/.socrates
REDIS_URL=redis://localhost:6379
ENABLE_RATE_LIMITING=false  # Disable for testing
LOG_LEVEL=DEBUG
DEBUG=true
ALLOWED_ORIGINS=http://localhost:3000,http://localhost:5173
```

**Start with:**
```bash
python socrates.py --full
```

### Staging Configuration

```bash
ENVIRONMENT=staging
JWT_SECRET_KEY=unique-staging-secret-key
ANTHROPIC_API_KEY=sk-ant-...  # Staging key
SOCRATES_API_HOST=0.0.0.0  # Accept all (behind reverse proxy)
SOCRATES_API_PORT=8000
DATABASE_URL=postgresql://user:pass@staging-db/socrates  # Optional
REDIS_URL=redis://redis:6379
ENABLE_RATE_LIMITING=true
LOG_LEVEL=INFO
ALLOWED_ORIGINS=https://staging.example.com
```

### Production Configuration

```bash
ENVIRONMENT=production
JWT_SECRET_KEY=generate-with-openssl-rand-hex-32
ANTHROPIC_API_KEY=sk-ant-...  # Production key
SOCRATES_API_HOST=0.0.0.0  # Behind reverse proxy
SOCRATES_API_PORT=8000
DATABASE_URL=postgresql://...  # PostgreSQL required
DATABASE_REPLICA_URLS=postgresql://...  # Optional replicas
REDIS_URL=redis://redis-cluster/0  # Redis cluster
SECURITY_DATABASE_ENCRYPTION=true  # Enable encryption
ENABLE_RATE_LIMITING=true
ENABLE_METRICS=true  # For monitoring
LOG_LEVEL=WARNING  # Less verbose
DEBUG=false
ALLOWED_ORIGINS=https://example.com,https://www.example.com
```

**Deploy with:**
```bash
docker run -e ENVIRONMENT=production \
           -e JWT_SECRET_KEY=... \
           -e ANTHROPIC_API_KEY=... \
           socrates:latest
```

## Setup Instructions

### Automatic Setup (Development)

```bash
python setup_env.py
```

This script:
- Generates `.env` file
- Creates JWT_SECRET_KEY
- Sets defaults for development
- Creates data directory

### Manual Setup

1. **Copy template**
   ```bash
   cp .env.example .env
   ```

2. **Generate JWT secret**
   ```bash
   # macOS/Linux
   openssl rand -hex 32

   # Windows (PowerShell)
   [Convert]::ToHexString((1..32 | ForEach-Object { Get-Random -Maximum 256 })) -join ''
   ```

3. **Get Anthropic API key**
   - Visit https://console.anthropic.com/
   - Create new API key
   - Copy to .env

4. **Update .env**
   ```bash
   nano .env  # or your editor
   ```

5. **Verify setup**
   ```bash
   python socrates.py --api  # Should start without errors
   ```

## Troubleshooting

### "No such file or directory: '.env'"

**Solution:**
```bash
python setup_env.py
# OR
cp .env.example .env
```

### "JWT_SECRET_KEY not set"

**Solution:**
```bash
# Generate and add to .env
openssl rand -hex 32 >> .env
```

### "Port 8000 already in use"

**Solutions:**
- Change `SOCRATES_API_PORT=8001`
- Kill other process: `lsof -i :8000`
- Let auto-detect: Don't set port (tries 8000, 8001, 8002, etc.)

### "Redis connection refused"

**Solution - Option 1:** Start Redis
```bash
redis-server
```

**Solution - Option 2:** Disable rate limiting
```bash
ENABLE_RATE_LIMITING=false
```

### "ANTHROPIC_API_KEY not set"

**Solution:**
- For code generation/analysis: Set API key
- For other features: Can work without it
- Leave empty for offline mode

### "Database locked" errors

**Solution:**
- Reduce concurrent connections
- Or use PostgreSQL (supports better concurrency)

## Environment Variable Precedence

Variables are resolved in this order (highest to lowest priority):

1. **Command-line arguments** (if tool supports them)
2. **Environment variables** (export VAR=value)
3. **.env file** (loaded by application)
4. **Defaults** (hardcoded in code)

Example:
```bash
# This overrides .env
export SOCRATES_API_PORT=9000

# Or inline with command
SOCRATES_API_PORT=9000 python socrates.py --api
```

## Security Considerations

### Secrets Management

**Never commit to version control:**
- `.env` file
- `JWT_SECRET_KEY`
- `ANTHROPIC_API_KEY`
- `DATABASE_ENCRYPTION_KEY`

**Use for production:**
- Vault (HashiCorp)
- AWS Secrets Manager
- Google Cloud Secret Manager
- Azure Key Vault
- Environment variables on deployment platform

### JWT_SECRET_KEY Security

- **Minimum 32 characters** in production
- **Unique per deployment**
- **Rotate regularly** (breaks existing tokens)
- **Generate with:** `openssl rand -hex 32`

### Database Security

- **Use HTTPS/TLS** for PostgreSQL connections
- **Enable encryption** with `SECURITY_DATABASE_ENCRYPTION=true` if sensitive
- **Use separate database user** with minimal permissions
- **Change default passwords** (if any)

### CORS Configuration

- **Development:** localhost allowed
- **Production:** Only your domain
- **Never:** Allow `*` in production
- **Example:** `https://example.com,https://www.example.com`

### For Development

```bash
ENABLE_RATE_LIMITING=false
ENABLE_METRICS=false

```

### For Production

```bash

DATABASE_POOL_SIZE=20  # Increase for high concurrency
```

### Database Optimization

```bash
# For PostgreSQL high-traffic
DATABASE_POOL_SIZE=30
DATABASE_MAX_OVERFLOW=50
DATABASE_URL=postgresql://user:pass@host/db?pool_size=30&max_overflow=50

# For large datasets
DATABASE_REPLICA_URLS=postgresql://user:pass@replica1/db,postgresql://user:pass@replica2/db
```

## Migration to PostgreSQL

When ready to migrate from SQLite to PostgreSQL:

1. **Create PostgreSQL database**
   ```bash
   createdb socrates
   ```

2. **Update .env**
   ```bash
   DATABASE_URL=postgresql://user:password@localhost:5432/socrates
   ```

3. **Run migration**
   ```bash
   python scripts/migrate_to_postgres.py
   ```

4. **Verify migration**
   ```bash
   python socrates.py --api
   # Both SQLite and PostgreSQL work during transition
   ```

5. **Switch to PostgreSQL-only**
   - Update `DATABASE_URL` in deployment
   - Remove SQLite database once verified
   - Can rollback if issues (SQLite still available)

## Best Practices

1. **Use setup_env.py for development**
   - Auto-generates sensible defaults
   - Saves time

2. **Version control .env.example, not .env**
   - .env.example shows all options
   - .env contains secrets

3. **Use environment-specific configs**
   - development: Permissive, logging verbose
   - staging: Close to production, testing enabled
   - production: Secure, minimal logging

4. **Document your environment**
   - What each variable does
   - Why it's set to that value
   - How to change it

5. **Validate on startup**
   - Check required variables present
   - Validate formats
   - Test connections

6. **Use strong secrets**
   - Minimum 32 characters
   - Random generation (openssl, etc.)
   - Unique per deployment

7. **Monitor configuration**
   - Log what config is used
   - Alert on missing required values
   - Track secrets rotation
