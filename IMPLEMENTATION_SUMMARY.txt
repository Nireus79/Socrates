================================================================================
                           HOW IT WORKS NOW
================================================================================

USER SETS API KEY:
  1. User provides API key via /llm/api-key endpoint
  2. Request reaches orchestrator with action="add_api_key"
  3. Handler validates and saves to database
  4. Response: {"status": "success"}

USER GETS QUESTION:
  1. User requests question via GET /projects/{id}/chat/question
  2. Backend looks up user's API key from database
  3. IF key found:
       - Create per-user LLMClient with their key
       - Questions use their API quota
     ELSE:
       - Use server's global API key
       - Questions use server's quota
  4. Questions are Claude-generated, specific to their project
  5. Response contains question text

USER REMOVES KEY:
  1. User submits DELETE /llm/api-key/anthropic
  2. Handler removes key from database
  3. Response: {"status": "success"}
  4. Next request falls back to server key automatically
